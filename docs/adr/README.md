# Architecture Decision Records

Architecture Decision Records capture important technical or workflow decisions
that deserve durable context.

Use an ADR when a decision materially affects:

- repository structure
- public interfaces
- contract or service boundaries
- trust or security assumptions
- contributor workflow expectations

## Naming

Use a simple numeric prefix and short slug, for example:

- `0001-contract-workspace-layout.md`
- `0002-request-and-verdict-schema-boundaries.md`

## Template

Start new ADRs from [0000-template.md](0000-template.md).
