# Support

## Where To Ask Questions

For now, use GitHub issues for project-related questions and general
contribution guidance.

If you are unsure whether something is a bug or a question, start with the
question issue template and include the context you have.

## When To Open A Bug Report

Open a bug report when:

- something behaves differently from what the repo currently documents
- a reproducible workflow is broken
- an example or setup step no longer works as described

Include:

- what you expected
- what happened instead
- how to reproduce it
- any relevant logs, screenshots, or environment details

## When To Open A Feature Request

Open a feature request when:

- you have a clear problem the project should solve
- you can explain the user or developer need
- the request is not already covered by an open issue

Try to describe the problem and constraints, not just the preferred solution.

## Design Discussions

If your question is really about a broader technical direction:

- open an issue with the question template
- explain the tradeoff or uncertainty clearly
- keep the first message narrow enough for others to respond productively

## Security Issues

Do not use public issues for security vulnerabilities. Follow
[SECURITY.md](SECURITY.md) instead.

## Current Project Reality

The project is still early. That means:

- some documentation may move as the repo evolves
- some implementation details may still be unsettled
- small clarifications and setup improvements are especially valuable
