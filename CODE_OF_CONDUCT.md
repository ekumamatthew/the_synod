# Code Of Conduct

## Our Commitment

We want The Synod to be a project where people can collaborate respectfully,
learn in public, and contribute safely regardless of experience level,
background, identity, or communication style.

We expect maintainers, contributors, and participants in project spaces to act
with professionalism, curiosity, and respect.

## Expected Behavior

Examples of behavior that help create a positive environment include:

- showing respect for different viewpoints and levels of experience
- giving actionable, technically grounded feedback
- assuming good intent while still being precise about problems
- accepting feedback gracefully
- focusing discussion on the work rather than on the person
- helping first-time contributors navigate the project

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, intimidation, or discrimination
- insulting or demeaning comments
- personal attacks or sustained hostility
- sexualized language or unwelcome attention
- publishing private information without permission
- deliberately disruptive behavior in issues, pull requests, or other project
  spaces

## Scope

This code of conduct applies in:

- GitHub issues
- pull requests
- project discussions
- any public or private communication spaces used to coordinate work on the
  project

## Reporting

If you experience or witness unacceptable behavior, report it privately to the
maintainers instead of escalating in public.

Until a dedicated reporting address exists, open a private security-style report
using the contact details in [SECURITY.md](SECURITY.md) or contact the project
owner directly through the repository owner account.

Please include:

- what happened
- where it happened
- who was involved, if known
- any screenshots, links, or other context that may help

## Maintainer Responsibilities

Maintainers are responsible for:

- responding in good faith to reports
- protecting reporters from retaliation where possible
- taking proportionate action based on the available facts
- communicating outcomes where appropriate and safe to do so

Possible responses may include warnings, content removal, temporary exclusion
from project spaces, or permanent bans in severe or repeated cases.
